Expressions

Expressions and Predicates can then be used to create the variousEnterprise Integration Patternsin the DSL or XML Configuration like the Recipient List.
To support dynamic rules Camel supports pluggableExpressionstrategies using a variety of different Languages.

API

If you are outside of the DSL and want to create your own expressions you can either implement theExpression interface, reuse one of the other builders or try theExpressionBuilder class.

Expression

The API for a Camel Expression is defined in theorg.apache.camel.Expressioninterface as shown:

public interface Expression {

    /**
     * Returns the value of the expression on the given exchange
     *
     * @param exchange the message exchange on which to evaluate the expression
     * @param type the expected type of the evaluation result
     * @return the value of the expression
     */
    <T> T evaluate(Exchange exchange, Class<T> type);
}
Predicate

The API for a Camel Predicate is defined in theorg.apache.camel.Predicateinterface as shown:

public interface Predicate {

    /**
     * Evaluates the predicate on the message exchange and returns true if this
     * exchange matches the predicate
     *
     * @param exchange the message exchange
     * @return true if the predicate matches
     */
    boolean matches(Exchange exchange);

}

Using Expressions in your IDE

To use different expression and predicates in your IDE you need to perform a static import of the builder class for the language(s) you wish to use.
